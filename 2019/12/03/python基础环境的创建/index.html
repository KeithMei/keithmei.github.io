1、python基础–运行环境的创建

python2版本， 虽然说2020年就停止更新。 但有些系统依然只是支持它，众多内网环境， 版本过低的系统， 上面也是安装着python2的版本。

所以为了避免环境之间产生的冲突 ， 故因此搭建一个虚拟的环境， 当然此套虚拟环境python2 与python3 中都支持。但安装包的名称以及一些命令两版本间有差别。详情可自行网上查看相应的叙述。

以下是安装步骤：

测试环境：
deepin系统（该命令也适用于ubuntu以及debian等系统。但不支持centos，redhat系统）
python版本：python2.7.13， python3.5.3
没有pip
在此使用的python版本是3。

1.1 基本环境的安装

一般系统都装有python2 与python3 版本的python运行环境， 若想要升级， 也可以源码升级或命令升级（详情可自行网上搜索）。

首先安装pip命令
apt-get install python3-pip

然后再安装虚拟包
pip3 install virtualenv

检查是否安装成功
virtualenv –version

1.2 创建一个vritulenv环境

整理一个目录用于存放virtualenv环境
mkdir /data/pyProjects
cd /data/pyProjects

创建这个目录的独立第三方库安装环境， 命名为 venv
virtualenv –no-site-packages venv

查看环境
cd venv
ls venv/
bin include lib

进入虚拟环境命令
source ./venv/bin/activate

因为是使用python3安装的virualenv， 所以自带的python版本是3

安装模块包
pip3 install django

退出虚拟环境
deactivate

1.3 其他用法

指定python版本创建虚拟环境
virtualenv -p /usr/bin/python py2venv

mkdir /data/secondProjects
这里的/usr/bin/python 必须是本机的python2版本

virtualenv常用参数：
继承系统三方库 –system-site-packages
添加参数–system-site-packages，项目检索库的时候，也会到系统的三方库中找
不添加时，默认只到虚拟环境中查找库

–no-site-packages参数，就是不带任何python环境下的第三方库过来

删除虚拟环境
直接删除虚拟环境所在目录即可删除整个环境

使用场景：
项目交接
场景：确保共享的项目能够在其他电脑上正常运行
解决：
方案一：
连同虚拟环境和项目一起拷贝给他人
方案二：
在虚拟环境中，冻结依赖需求文本
把项目和依赖需求文本给他人
他人在本地创建一个新的虚拟环境，并根据依赖需求文本安装相关库

技能点：
冻结项目需求文本 pip3 freeze > requirements.txt
根据需求文本，安装项目依赖库 pip3 install -r requirements.txt （在激活的虚拟环境中）
示例操作如下：
pip3 freeze > requirements.txt

1.4 虚拟环境中pip下载模块速度慢， 可以更改国内的pip模块源

新版ubuntu要求使用https源，要注意。
清华：https://pypi.tuna.tsinghua.edu.cn/simple
阿里云：http://mirrors.aliyun.com/pypi/simple/
中国科技大学 https://pypi.mirrors.ustc.edu.cn/simple/
华中理工大学：http://pypi.hustunique.com/
山东理工大学：http://pypi.sdutlinux.org/
豆瓣：http://pypi.douban.com/simple/

临时更改：
加参数-i https://pypi.tuna.tsinghua.edu.cn/simple

pip install -i https://pypi.tuna.tsinghua.edu.cn/simple pyspider
这样就会从清华这边的镜像去安装pyspider库

永久修改：
Linux下，修改 ~/.pip/pip.conf (没有就创建一个文件夹及文件。文件夹要加“.”，表示是隐藏文件夹)
内容如下：
[global]
index-url = https://pypi.tuna.tsinghua.edu.cn/simple
[install]
trusted-host=mirrors.aliyun.com
